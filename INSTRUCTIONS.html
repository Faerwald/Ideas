Get this Portfolio Online + Protect Your IP

Overview: Host the site on GitHub Pages, store PDFs in Google Drive, add rights notices to the site and each PDF, timestamp with OpenTimestamps, and mint DOIs via Zenodo.

1) Publish the site

  1. Create a public repo named YOUR_USERNAME.github.io on GitHub.
  2. Upload all files from this folder. Commit.
  3. Your site will be at https://YOUR_USERNAME.github.io/.

2) Link PDFs from Google Drive

  1. Upload your PDFs to Drive, set sharing to Anyone with the link: Viewer.
  2. Get the file ID from the Drive URL and place it into papers.json as driveId.
  3. Optionally add a direct download: the site builds this from the ID automatically.

3) Rights notice in each PDF

  1. Decide license terms (e.g., “All rights reserved” or a CC license).
  2. Use scripts/append_copyright_page.py to append a one-page notice to each PDF (see script header for instructions).

4) Timestamp files with OpenTimestamps

Install locally:

pip install opentimestamps-client
ots stamp MyPaper.pdf
ots upgrade MyPaper.pdf.ots   # (optional, later)
ots verify MyPaper.pdf.ots

5) Mint DOIs with Zenodo (via GitHub releases)

  1. Sign in at Zenodo with GitHub and authorize.
  2. Toggle your repository “on” in Zenodo’s GitHub section.
  3. Create a GitHub Release (v0.1.0). Zenodo will archive it and mint a DOI.
  4. Add the DOI badge/link back into your repo README and the site.

6) Add a site-wide licensing page

Edit LICENSE.html. Typical options:

  • All rights reserved (default). Readers can view; ask permission for reuse.
  • Creative Commons variants (e.g., CC BY-NC-ND 4.0) for noncommercial, no-derivatives sharing. Put the chosen license line on the PDF notice page and in LICENSE.html.

7) Patents (if applicable)

Public disclosure may affect patent rights outside the U.S. U.S. has a 1-year grace period; EPO generally no broad grace period. If patenting is possible, file first or consult counsel before posting.

8) Maintenance workflow

  1. Upload new PDF to Drive → add to papers.json.
  2. Run the append script (for rights page) and ots stamp to generate an .ots proof.
  3. Commit and push updates to GitHub Pages. Optionally cut a release to mint a new DOI.
